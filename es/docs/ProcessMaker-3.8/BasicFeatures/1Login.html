ProcessMaker 3.8 - Iniciar Sesión

Iniciar Sesión​

Puedes iniciar sesión en ProcessMaker de diferentes maneras dependiendo de si accedes a través de la interfaz estándar o mediante un complemento empresarial.

Consulta las siguientes secciones que explican los distintos métodos para iniciar sesión en ProcessMaker:

  • Interfaz estándar: Iniciar sesión en ProcessMaker a través de la interfaz estándar.
  • Autenticación de Google: Usa el complemento de autenticación de Google, disponible para la versión 3.3 y posteriores de ProcessMaker. Ten en cuenta que tu organización debe haber implementado este complemento.
  • Autenticación SAML: Usa el complemento de autenticación SAML, disponible para la versión 3.4.0 y posteriores de ProcessMaker. Tu organización debe haber implementado este complemento.

Iniciar Sesión en ProcessMaker a través de la Interfaz Estándar​

Sigue estas instrucciones para iniciar sesión en ProcessMaker a través de la interfaz estándar:

  1. Ingresa la dirección IP (o nombre de dominio) y el número de puerto del servidor o instancia que aloja ProcessMaker en un navegador web.

    http://<DIRECCIÓN-IP>:<PUERTO>

    Por ejemplo, si ProcessMaker se ejecuta en la dirección IP 192.168.1.110 en el puerto 3018, introduce:

    http://192.168.1.110:3018

    Si solo se incluye la dirección IP (o el nombre de dominio) en la URL, el navegador redirige automáticamente a la pantalla de inicio de sesión de ProcessMaker:

    http://<DIRECCIÓN-IP>:<PUERTO>/sys/es/neoclassic/login/login.html

    Aparece la pantalla de inicio de sesión.

  2. Realiza una de las siguientes acciones:

    a. Si ProcessMaker fue recién instalado, ingresa las siguientes credenciales para iniciar sesión como administrador:

    • Usuario: admin
    • Contraseña: admin
    • Espacio de trabajo: workflow

    Una vez que hayas iniciado sesión como el usuario "admin", crea nuevos usuarios en ProcessMaker. Durante tu primer inicio de sesión, asegúrate de cambiar la contraseña del administrador para mayor seguridad. Después de crear los usuarios, podrán iniciar sesión con sus propios nombres de usuario y contraseñas.

    b. Si inicias sesión como usuario, ingresa tu nombre de usuario y contraseña. Si el espacio de trabajo no está incluido en la URL, selecciona el espacio de trabajo. El espacio de trabajo predeterminado es "workflow".

  3. Selecciona el idioma predeterminado para mostrar ProcessMaker si se han instalado otros idiomas. El idioma predeterminado es el inglés.

  4. Haz clic en Login.

Iniciar Sesión mediante el Complemento Empresarial de Autenticación de Google​

nota

Para iniciar sesión mediante la autenticación de Google, tu organización debe haber implementado el complemento Google Authentication Enterprise Plugin. Este complemento solo está disponible para la versión Enterprise de ProcessMaker 3.3.0 y posteriores.

Sigue estas instrucciones para iniciar sesión en ProcessMaker utilizando la autenticación de Google:

  1. Ingresa la dirección IP (o nombre de dominio), número de puerto, nombre del espacio de trabajo y el idioma predeterminado del espacio de trabajo en un navegador web estándar utilizando la siguiente URL:

    http://<DIRECCIÓN-IP>:<PUERTO>/sys<ESPACIO-DE-TRABAJO>/<IDIOMA-POR-DEFECTO>/neoclassic/login/login.html

    Por ejemplo, si ProcessMaker se ejecuta en la dirección IP 192.168.1.110 en el puerto 3018, utilizando el espacio de trabajo llamado BPMWORKSPACE, y el idioma por defecto es inglés, entonces introduce:

    http://192.168.1.110:3018/sysbpmworkspace/en/neoclassic/login/login.html

    Aparece la pantalla de inicio de sesión.

  2. Haz clic en Sign in with Google. Si ya has iniciado sesión en tu cuenta de Google, Google te autenticará y enviará tus credenciales a ProcessMaker. Si no has iniciado sesión en una cuenta de Google, consulta Iniciar sesión en Gmail.

En lugar de hacer clic en el botón Sign in with Google, también puedes iniciar sesión con tus credenciales de ProcessMaker como se describe en la sección Iniciar Sesión en ProcessMaker a través de la Interfaz Estándar.

Iniciar Sesión mediante el Complemento Empresarial de Autenticación SAML​

nota

Para iniciar sesión mediante la autenticación SAML, tu organización debe haber implementado el complemento SAML Authentication Enterprise Plugin. Este complemento solo está disponible para la versión Enterprise de ProcessMaker 3.4.0 y posteriores.

Sigue estas instrucciones para iniciar sesión en ProcessMaker utilizando la autenticación SAML:

  1. Ingresa la dirección IP (o nombre de dominio), número de puerto, nombre del espacio de trabajo y el idioma predeterminado del espacio de trabajo en un navegador web estándar utilizando la siguiente URL:

    http://<DIRECCIÓN-IP>:<PUERTO>/sys<ESPACIO-DE-TRABAJO>/<IDIOMA-POR-DEFECTO>/neoclassic/login/login.html

    Por ejemplo, si ProcessMaker se ejecuta en la dirección IP 192.168.1.110 en el puerto 3018, utilizando el espacio de trabajo llamado BPMWORKSPACE, y el idioma por defecto es inglés, entonces introduce:

    http://192.168.1.110:3018/sysbpmworkspace/en/neoclassic/login/login.html

  2. La URL redirige al servicio IdP (en este caso, Okta). Dependiendo de la configuración de Field Matching, la pantalla de inicio de sesión se mostrará.

    En los campos Usuario y Contraseña, ingresa tus credenciales de Okta.

  3. Haz clic en Sign in para iniciar sesión en ProcessMaker.

precaución

Si ocurre algún problema al intentar iniciar sesión en ProcessMaker, se mostrará el siguiente mensaje de error:

  • Cuando no se encuentra el registro de usuario
    Error de autenticación SAML: El registro de usuario no fue encontrado, por favor verifica la configuración con tu administrador del sistema.

  • Cuando hay más de un registro coincidente
    Error de autenticación SAML: Se encontraron múltiples registros que coinciden con la descripción, por favor contacta a tu administrador del sistema para corregir esta configuración.

ProcessMaker almacena todos los errores en el registro estándar de ProcessMaker.

El registro estándar de ProcessMaker guarda los intentos de inicio de sesión mediante SAML con el nivel de severidad INFO, junto con un mensaje indicando que un usuario ha iniciado sesión utilizando SAML SSO.

Si es necesario deshabilitar la autenticación SAML en el espacio de trabajo, inicia sesión utilizando tus credenciales de ProcessMaker como se describe en la sección Iniciar Sesión en ProcessMaker a través de la Interfaz Estándar.

Configurar los Ajustes de Inicio de Sesión​

Esta sección explica cómo configurar los ajustes de inicio de sesión que afectan a la interfaz estándar.

Para obtener información sobre cómo configurar los ajustes de:

  • El complemento de autenticación de Google, que permite a los usuarios iniciar sesión con sus credenciales de Google, consulta la sección Complemento de Autenticación de Google.
  • El complemento de autenticación SAML, que permite a los usuarios iniciar sesión con sus credenciales de un proveedor SAML, consulta la sección Complemento de Autenticación SAML.
nota

Los usuarios deben tener el permiso PM_SETUP_LOGIN asignado a su rol para acceder a ADMIN > Configuración > Inicio de Sesión.

Consulta las siguientes secciones sobre la configuración de inicio de sesión:

  • Acceder a la pantalla de configuración de inicio de sesión.
  • Establecer el idioma predeterminado en la pantalla de inicio de sesión.
  • Habilitar la recuperación de contraseña.

Acceder a la Pantalla de Configuración de Inicio de Sesión​

Para acceder a los ajustes de inicio de sesión, ve a ADMIN > Configuración > Inicio de Sesión. Aparecerá la pantalla de configuración de inicio de sesión.

Establecer el Idioma Predeterminado en la Pantalla de Inicio de Sesión​

Utiliza la opción Idioma Predeterminado para seleccionar el idioma que se mostrará por defecto en la pantalla de inicio de sesión. ProcessMaker está disponible por defecto solo en inglés, pero se pueden instalar archivos de traducción para otros idiomas. Esta configuración debe establecerse para cada espacio de trabajo.

Para establecer el idioma predeterminado en la pantalla de inicio de sesión de un espacio de trabajo, sigue estos pasos:

  1. Accede a los ajustes de inicio de sesión.

  2. Realiza una de las siguientes acciones:

    • Selecciona el idioma predeterminado para ese espacio de trabajo en el menú desplegable Idioma Predeterminado.
    • Selecciona Usar el idioma de la URL para permitir que la configuración del idioma en la URL determine el idioma predeterminado. Por ejemplo, la URL de inicio de sesión podría contener "en" para inglés o "es" para español.

  3. Haz clic en Guardar Configuración.

Habilitar la Recuperación de Contraseña​

Si habilitas esta opción, cuando un usuario olvide su contraseña de ProcessMaker, aparecerá un enlace en la pantalla de inicio de sesión para enviar una nueva contraseña generada aleatoriamente a su correo electrónico. Esta contraseña aleatoria permitirá al usuario crear una nueva contraseña.

Para habilitar la opción de recuperación de contraseña, marca la opción Habilitar Recuperación de Contraseña y luego haz clic en Guardar Configuración.

La próxima vez que un usuario intente iniciar sesión en ProcessMaker, aparecerá el enlace ¿Olvidaste tu contraseña? en la pantalla de inicio de sesión.

Un usuario que olvidó su contraseña de ProcessMaker debe seguir estos pasos para crear una nueva contraseña:

  1. Haz clic en ¿Olvidaste tu contraseña? Aparecerá un formulario para ingresar el nombre de usuario y el correo electrónico donde se enviará la contraseña generada aleatoriamente.

  2. El usuario debe proporcionar la siguiente información:

    • Ingresar su nombre de usuario en el campo Usuario.
    • Ingresar su correo electrónico donde se enviará la contraseña en el campo Email.

  3. Haz clic en Enviar Solicitud. Si el nombre de usuario y el correo electrónico coinciden con un registro de usuario en ProcessMaker, se enviará un correo con una nueva contraseña generada aleatoriamente.

nota

Para recibir el correo de recuperación de contraseña, primero ProcessMaker debe estar configurado para conectarse a un servidor de correo. Si no lo está, se mostrará un mensaje de advertencia.

Habilitar la Recuperación de Contraseña​

Al habilitar esta opción, si un usuario olvida la contraseña para iniciar sesión en ProcessMaker, se mostrará un enlace en la pantalla de inicio de sesión para enviar una nueva contraseña generada aleatoriamente a la dirección de correo electrónico del usuario. Esta contraseña temporal le permitirá crear una nueva.

Para permitir que los usuarios obtengan nuevas contraseñas, marca la opción Habilitar Recuperación de Contraseña y luego haz clic en Guardar Configuración.

La próxima vez que un usuario inicie sesión en ProcessMaker, el enlace ¿Olvidaste tu contraseña? aparecerá en la pantalla de inicio de sesión.

Un usuario que haya olvidado su contraseña de inicio de sesión en ProcessMaker debe seguir estos pasos para generar una nueva:

  1. Haz clic en ¿Olvidaste tu contraseña? Se mostrará un formulario para ingresar el nombre de usuario y la dirección de correo electrónico donde se enviará la nueva contraseña generada aleatoriamente.

  2. El usuario debe proporcionar la siguiente información:

    • Nombre de usuario en el campo Usuario.
    • Correo electrónico en el campo Email para recibir la contraseña generada.

  3. Haz clic en Enviar solicitud. Si el nombre de usuario y el correo electrónico coinciden con un registro de usuario en ProcessMaker, se enviará un correo electrónico con una nueva contraseña generada aleatoriamente.

nota

Para recibir el correo electrónico de recuperación de contraseña, ProcessMaker debe estar configurado para conectarse a un servidor de correo electrónico. Si no está configurado, se mostrará un mensaje de advertencia.

  1. Ingresa el nombre de usuario y la nueva contraseña enviada por correo electrónico.

  2. Haz clic en Iniciar sesión. Se mostrará una pantalla para que restablezcas tu contraseña.

Cambiar la Página de Destino Después de Iniciar Sesión​

Después de iniciar sesión, los usuarios con los roles PROCESSMAKER_OPERATOR y PROCESSMAKER_MANAGER son redirigidos por defecto al menú INICIO, donde se muestra la lista de casos del usuario. Esta pantalla está ubicada en la siguiente dirección:

http://<DIRECCIÓN-IP>:<PUERTO>/sys<ESPACIO-DE-TRABAJO>/<IDIOMA>/<SKIN>/cases/main

Los usuarios con el rol PROCESSMAKER_ADMIN o con el permiso PM_FACTORY en su rol son redirigidos por defecto al menú DISEÑADOR después de iniciar sesión. Esta pantalla está ubicada en la siguiente dirección:

http://<DIRECCIÓN-IP>:<PUERTO>/sys<ESPACIO-DE-TRABAJO>/<IDIOMA>/<SKIN>/processes/main

Cada usuario que tenga el permiso PM_EDITPERSONALINFO en su rol puede personalizar la pantalla a la que ProcessMaker lo dirige después de iniciar sesión. Para hacerlo:

  1. Después de iniciar sesión, haz clic en el nombre de usuario en la esquina superior derecha para acceder a tu perfil.

  2. Haz clic en Editar en la parte inferior derecha para modificar el perfil.

  3. En la sección Preferencias, selecciona el menú predeterminado en la opción Menú Principal por Defecto. En el menú INICIO, la lista de casos en la Bandeja de Entrada se muestra por defecto. Para cambiar qué opción se selecciona en la barra lateral de Casos, selecciona una opción en el menú desplegable Opción de Menú de Casos por Defecto.

También puedes cambiar la página de destino después de iniciar sesión especificando la dirección de la siguiente pantalla en la URL de inicio de sesión:

http://<DIRECCIÓN-IP>:<PUERTO>/sys<ESPACIO-DE-TRABAJO>/<IDIOMA>/<SKIN>/login/login?u=<SIGUIENTE-DIRECCIÓN>

No incluyas la dirección IP en <SIGUIENTE-DIRECCIÓN> y asegúrate de que la dirección comience con una / (barra diagonal).

Por ejemplo, si deseas ir directamente al menú ADMINISTRADOR después de iniciar sesión en el servidor de ProcessMaker en el ejemplo anterior:

http://192.168.1.50/sysaccounting/fr/company/login/login.html?u=/sysaccounting/fr/company/setup/main

Registros de Inicio de Sesión​

Cada vez que un usuario inicia sesión en ProcessMaker, se guarda un registro en la tabla **wf_<ESPACIO-DE-TRABAJO>.LOGIN_LOG** en la base de datos. Esta tabla contiene los siguientes campos:

NombreDescripción
LOG_UIDUn ID único para identificar la sesión de inicio de sesión.
LOG_STATUSEstado de la sesión de inicio de sesión, puede ser "ACTIVO" o "CERRADO". Si está en "CERRADO", el usuario finalizó la sesión haciendo clic en Cerrar sesión. Si está en "ACTIVO", la sesión puede seguir abierta o puede haber expirado por inactividad.
LOG_IPDirección IP del dispositivo del usuario. Si el usuario inició sesión desde el mismo equipo donde está instalado ProcessMaker, el valor será 127.0.0.1.
LOG_SIDID de sesión que PHP usa para identificar de manera única la sesión. Tiene 26 caracteres compuestos de letras y números en minúsculas.
LOG_INIT_DATEFecha y hora en que comenzó la sesión de inicio de sesión en formato YYYY-MM-DD HH:MM:SS.
LOG_END_DATEFecha y hora en que la sesión de inicio de sesión finalizó en formato YYYY-MM-DD HH:MM:SS. Este campo solo se establece si el usuario hace clic en Cerrar sesión. Si la sesión sigue activa o expiró por inactividad, el valor será NULL.
LOG_CLIENT_HOSTNAMEDirección IP y número de puerto del servidor de ProcessMaker.
USR_UIDID único del usuario que inició sesión.

Para generar informes sobre los registros de inicio de sesión de los usuarios, puedes consultar la tabla **wf_<ESPACIO-DE-TRABAJO>.LOGIN_LOG** con consultas SQL dentro de Dynaform, utilizando executeQuery() en Triggers o con herramientas de reportes externas como Jasper Reports, Pentaho Reporting o Crystal Reports.
